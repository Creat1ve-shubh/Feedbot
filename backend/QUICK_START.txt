# QUICK START - Virtual Environment Setup

## For Windows Users (Fastest)

### Option 1: PowerShell Script (Recommended)
```powershell
cd backend
.\setup.ps1
```

This will:
- Create virtual environment (venv/)
- Install all dependencies from requirements.txt
- Verify installation
- Activate the environment

The environment is automatically activated when done!

### Option 2: Python Script (Cross-Platform)
```powershell
cd backend
python setup.py
```

### Option 3: Manual Setup

1. Create virtual environment:
   ```powershell
   python -m venv venv
   ```

2. Activate it:
   ```powershell
   .\venv\Scripts\Activate.ps1
   ```

3. Install dependencies:
   ```powershell
   pip install -r requirements.txt
   ```

## For Linux/macOS Users

### Option 1: Bash Script (Recommended)
```bash
cd backend
chmod +x setup.sh
./setup.sh
```

### Option 2: Python Script
```bash
cd backend
python3 setup.py
```

### Option 3: Manual Setup

1. Create virtual environment:
   ```bash
   python3 -m venv venv
   ```

2. Activate it:
   ```bash
   source venv/bin/activate
   ```

3. Install dependencies:
   ```bash
   pip install -r requirements.txt
   ```

## What Gets Installed

From requirements.txt:
- fastapi (web framework)
- uvicorn (ASGI server)
- sqlalchemy (database ORM)
- psycopg2 (PostgreSQL driver)
- redis (cache client)
- celery (async tasks)
- pydantic (data validation)
- pandas, numpy, scikit-learn (data science)
- praw, tweepy (social media APIs)
- beautifulsoup4 (web scraping)
- httpx (HTTP client)
- And more...

## Verify Installation

After setup, check that packages are installed:

```bash
python -c "import fastapi; import sqlalchemy; import redis; print('SUCCESS: All core packages installed')"
```

## Next Steps

Once activated and dependencies installed:

1. Create database tables:
   ```bash
   python migrate.py
   ```

2. Verify everything is working:
   ```bash
   python verify_integration.py
   ```

3. Start the backend:
   ```bash
   docker-compose up
   # In another terminal:
   uvicorn app:app --reload
   ```

## Troubleshooting

### "Command not found: python"
Try: `python3` instead of `python`

### "Module not found" error
Make sure venv is activated:
- Windows: `.\venv\Scripts\Activate.ps1`
- Linux/macOS: `source venv/bin/activate`

### "pip is not installed"
Run: `python -m pip install --upgrade pip`

### "venv already exists"
Delete and recreate:
- Windows: `Remove-Item -Recurse venv`
- Linux/macOS: `rm -rf venv`

## Deactivate Virtual Environment

When done working:
```bash
deactivate
```

This returns you to the system Python.
